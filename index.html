Objective 1.1: OSI Model

Question 1

Which of the following steps of the troubleshooting methodology would most likely include checking through each level of the OSI model after the problem has been identified?

A. Establish a theory.

B. Implement the solution.

C. Create a plan of action.

D. Verify functionality.

Show Answer & Explanation ▶

Correct Answer: A. Establish a theory.

Why this is correct: The "Establish a theory" step involves making an educated guess about the problem's cause. The OSI model is a powerful tool for this. A common troubleshooting method is to form theories layer by layer (e.g., "Is it a physical cable issue?" - Layer 1 theory; "Is it an IP address issue?" - Layer 3 theory). This structured approach helps narrow down possibilities efficiently.

Why others are incorrect:

  • Implement the solution: This happens after you've tested your theory and confirmed the cause.
  • Create a plan of action: This step comes just before implementation, where you decide *how* to fix the problem.
  • Verify functionality: This is the final step, performed after the fix is applied to ensure the problem is resolved.

Key Concept: The OSI model provides a systematic framework for troubleshooting network issues by breaking down complex problems into manageable layers.

Question 2

Which of the following layers of the OSI model is responsible for end-to-end encryption?

A. Presentation

B. Application

C. Session

D. Transport

Show Answer & Explanation ▶

Correct Answer: A. Presentation

Why this is correct: The Presentation Layer (Layer 6) is responsible for data translation, formatting, and syntax. This includes key security functions like encryption and decryption (e.g., SSL/TLS). Its job is to ensure that data sent from one system's application layer is readable by the destination system's application layer, and encryption is a critical part of that process.

Why others are incorrect:

  • Application (Layer 7): Provides the user interface to the network (e.g., browser) but relies on lower layers for encryption.
  • Session (Layer 5): Manages the dialogue between devices but doesn't handle the data's format or encryption.
  • Transport (Layer 4): Manages end-to-end data delivery (e.g., TCP/UDP) but not its encryption.

Real-world Example: When you visit a secure website (HTTPS), the encryption happens at the Presentation Layer, even though your browser (Application Layer) initiates the connection.

Question 9

Which of the following technologies is the best choice to listen for requests and distribute user traffic across web servers?

A. Router

B. Switch

C. Firewall

D. Load balancer

Show Answer & Explanation ▶

Correct Answer: D. Load balancer

Why this is correct: A load balancer is specifically designed to distribute network traffic across multiple servers to ensure no single server becomes overwhelmed. It improves responsiveness and increases availability of applications.

Why others are incorrect:

  • Router: Connects different networks and routes traffic between them, but doesn't distribute traffic across servers.
  • Switch: Connects devices within the same network and forwards traffic based on MAC addresses.
  • Firewall: Provides security by filtering traffic but doesn't distribute load.

Practical Application: Large websites like Amazon or Netflix use load balancers to handle millions of requests by distributing them across thousands of servers worldwide.
